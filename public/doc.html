API Documentation

Endpoint [refresh]

URL: /api/refresh

Method: GET

Description

The "Refresh Users" endpoint allows you to refresh user data by specifying the country name. It ensures that the provided server key is valid before initiating the refresh process.

Authentication

Authentication is performed using the key request header, which should match the SERVER_KEY configured in the server environment.

Request Parameters

  • country (required): The name of the country for which you want to refresh user data.

Request Example

GET /api/refresh?key=your-server-key&country=YourCountryName

Response

The API returns a JSON response with the following possible elements:

Success Response

{
    "message": "All users data of YourCountryName has been refreshed"
}

Error Response

{
    "message": "Invalid API key provided"
}
{
    "message": "Please provide country name in query"
}
